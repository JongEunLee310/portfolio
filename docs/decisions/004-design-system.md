# ADR-004: 테마 토큰 기반 히어로와 라이트 콘텐츠 카드 디자인 시스템

## 상태

Accepted

## 결정

공통 디자인은 테마 토큰 기반 히어로, 라이트 콘텐츠 카드, 제한적인 포인트 컬러를 기준으로 구성한다.

## 규칙

- 모든 페이지 Hero는 현재 테마에 맞춰 라이트모드에서는 밝은 아이보리 그라데이션과 골드 포인트, 다크모드에서는 약한 네이비 그라데이션과 블루 포인트를 사용한다.
- Hero 배경과 포인트 글로우는 `src/styles/classNames.ts`의 `heroSurface`에서 관리한다.
- Footer는 다크 네이비 계열을 사용한다.
- 본문 목록과 카드 영역은 라이트 배경을 사용한다.
- CTA, 선택 상태, 핵심 강조 색은 하드코딩한 색상 대신 `--color-accent`, `--color-accent-bg`, `--color-accent-border` 토큰을 사용한다.
- 라이트모드에서 배너, 성과 지표, 개선 결과 카드에는 파란색 계열 장식을 사용하지 않는다.
- 반복 스타일은 `src/styles/classNames.ts` 또는 공통 컴포넌트로 분리한다.

## 예시 이미지 기반 페이지 규칙

사용자가 제공한 로컬 참고 이미지는 Git에 포함하지 않는다. 이미지는 `/Users/sleepyowl/Downloads`에만 두고, 구현에는 아래 관찰된 디자인 규칙만 반영한다.

### 공통

- Header는 다크 배경 위 고정형에 가깝게 보이게 하고, 활성 메뉴는 테마 포인트 색상과 하단 라인으로 구분한다.
- Hero는 왼쪽 메시지, 오른쪽 3D 아키텍처 비주얼의 2열 구성을 기본으로 한다.
- 다크 영역은 배경 패턴, 약한 전체 배경 그라데이션, 포인트 글로우를 사용하되, 본문 가독성을 해치지 않게 절제한다.
- 라이트 본문은 큰 흰색 카드와 얇은 테두리, 부드러운 그림자를 사용한다.
- 목록 필터 사이드바와 상세 TOC 레일은 역할은 다르지만 왼쪽 선, uppercase heading, `--color-accent` 기반 active/hover 상태를 공유한다.
- 목록 페이지의 Hero와 목록 툴바 사이에는 상단 여백을 둬 정렬 드롭다운과 보기 전환 버튼이 배너에 붙어 보이지 않게 한다.
- 목록 툴바의 정렬 드롭다운은 옵션 텍스트 길이와 관계없이 프로젝트 목록과 기술 노트 목록에서 같은 너비를 유지한다.
- Footer는 다크 배경, 연락 채널, 이력서 다운로드 CTA를 한 줄 그리드로 배치한다.

### Home

- 첫 화면은 강한 다크 Hero와 2개 CTA를 둔다.
- Featured Projects는 큰 가로 카드 2개를 사용하고, 썸네일과 기술 태그를 함께 보여준다.
- Other Projects는 작고 넓은 카드로 요약 정보를 빠르게 훑게 한다.
- Technical Notes Preview는 다크 섹션의 작은 카드 리스트로 리듬을 바꾼다.
- Tech Stack은 라이트 배경의 미니 카드 그리드로 마무리한다.

### Projects

- Hero 아래에 카테고리 pill 필터를 둔다.
- 목록 화면은 왼쪽 필터 사이드바와 오른쪽 3열 프로젝트 카드 그리드를 기본으로 한다.
- 필터 사이드바는 카드 그리드의 첫 행과 같은 세로 위치에서 시작하고, 정렬/보기 전환 툴바는 카드 그리드 위에 둔다.
- Hero 아래 목록 영역은 `pt-8 lg:pt-10` 수준의 상단 여백으로 배너와 툴바를 분리한다.
- 정렬 드롭다운은 고정 너비를 사용해 `대표 프로젝트순` 같은 긴 옵션이 있어도 기술 노트 목록의 드롭다운과 같은 크기로 보이게 한다.
- 필터 사이드바의 기술 스택 목록은 3개까지 접힌 상태로 시작하고, `더 보기`/`접기` 버튼으로 부드럽게 확장/축소한다. 확장 상태는 짧은 내부 스크롤 영역을 사용한다.
- 필터 사이드바의 버튼, 체크박스, 라디오 선택 상태는 `--color-accent` 등 테마 토큰을 사용해 light/dark 모드 모두에서 일관되게 보이게 한다.
- 카드에는 썸네일, 카테고리 배지, 제목, 요약, 태그, 상세/GitHub 버튼을 모두 포함한다.
- 정렬 버튼, 그리드/리스트 토글, 숫자 페이지네이션을 제공한다.
- 페이지네이션은 현재 페이지 주변 번호를 최대 5개만 표시해 모바일과 데스크톱에서 폭이 안정적이어야 한다.

### Project Detail

- Hero에는 프로젝트명, 한 줄 설명, 기간/역할/팀 규모, 핵심 성과 배지, 대표 대시보드 이미지를 배치한다.
- 상세 본문은 프로젝트 개요, 문제 정의, 해결 방향, 아키텍처, 핵심 기능, 스크린샷, 기술 스택, 기여, 성과, 트러블슈팅, 회고 순서를 따른다.
- 데스크톱 본문에는 왼쪽 sticky 목차를 얇은 레일 형태로 배치해 케이스 스터디 섹션 이동을 돕고, 모바일에서는 본문 집중을 위해 숨긴다.
- 아키텍처는 넓은 다이어그램 카드로 처리하고, 성과 지표는 라이트 카드와 테마 포인트 색상으로 강조한다.
- 성과 지표와 개선 결과 카드는 `--color-accent` 계열 토큰을 사용해 라이트모드에서는 골드, 다크모드에서는 블루로 보이게 한다.

### Technical Notes

- 목록 페이지는 Hero 다음에 카테고리 pill 필터를 둔다.
- 추천 글은 큰 가로 카드로 강조하고, 일반 글은 3열 카드 그리드를 사용한다.
- 필터 사이드바는 카드 그리드의 첫 행과 같은 세로 위치에서 시작하고, 정렬/보기 전환 툴바는 카드 그리드 위에 둔다.
- Hero 아래 목록 영역은 프로젝트 목록과 같은 상단 여백을 사용해 배너와 툴바 사이의 리듬을 맞춘다.
- 정렬 드롭다운은 프로젝트 목록과 같은 고정 너비를 사용해 두 목록 페이지의 툴바 크기를 일관되게 유지한다.
- 카드에는 카테고리 배지, 썸네일, 제목, 요약, 날짜, 읽는 시간을 포함한다.
- 필터 사이드바의 태그 체크박스와 관련 프로젝트 체크박스는 각각 3개까지 접힌 상태로 시작하고, `더 보기`/`접기` 버튼으로 부드럽게 확장/축소한다. 확장 상태는 짧은 내부 스크롤 영역을 사용한다.
- 필터 사이드바의 태그 체크박스, 관련 프로젝트 체크박스, `더 보기`/`접기` 버튼은 `--color-accent` 등 테마 토큰을 사용해 light/dark 모드별 선택 색상을 구분한다.
- 프로젝트 목록과 동일한 숫자 페이지네이션 패턴을 사용하며, 표시 번호는 현재 페이지 주변 최대 5개로 제한한다.

### Technical Note Detail

- Hero에는 글 제목, 카테고리, 날짜, 읽는 시간, 작성자, 태그를 둔다.
- 데스크톱 본문은 프로젝트 상세와 같은 얇은 왼쪽 sticky 레일과 오른쪽 글 본문 2열 구성을 사용한다.
- 왼쪽 레일에는 목차와 관련 글 링크만 배치하고, 배너에 이미 노출된 태그/기술 목록은 반복하지 않는다.
- 코드블록은 다크 배경으로, 성능 비교는 숫자 중심의 카드로 보여준다.
- 글 하단에는 관련 프로젝트와 전체 기술 노트 목록 CTA를 둔다.

### About

- Hero는 개발자 정체성과 프로젝트/연락 CTA를 제공한다.
- 본문 첫 섹션은 프로필 카드와 소개 카드를 조합한다.
- 강점, 경력/교육, 기술 스택, 일하는 방식, 지표 기반 성장, CTA 순서로 구성한다.
- 프로필 이미지는 실제 사진 또는 일러스트를 사용하되, 임시 아바타는 나중에 교체한다.

### Contact

- Hero는 협업 제안 메시지와 연락 관련 3개 가치 배지를 포함한다.
- 본문은 문의 폼 카드와 연락처 정보 카드를 2열로 배치한다.
- 연락처 정보에는 Email, GitHub, LinkedIn, GitLab, Resume 다운로드를 제공한다.
- FAQ는 아코디언 형태를 기준으로 하고, 하단 CTA는 다크 배너로 구성한다.
